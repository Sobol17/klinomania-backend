# API

Публичный контракт мобильного API Klinomania хранится в `docs/api/openapi.yaml`. Markdown-файлы в этой папке поясняют сценарии, правила авторизации, ошибки и бизнес-поведение endpoints.

## Базовые правила

- Базовый префикс текущей версии: `/api/v1`.
- Формат данных: JSON.
- Защищенные endpoints используют `Authorization: Bearer <token>`.
- Token выдается через Laravel Sanctum после успешной авторизации клиента или клинера.
- Любое изменение request/response/status codes должно обновлять `docs/api/openapi.yaml` в том же изменении кода.

## Разделы

- `openapi.yaml` - машинно-читаемый контракт API.
- `auth.md` - авторизация клиента, клинера и текущие dev-заглушки.
- `services.md` - каталог услуг.
- `orders.md` - клиентские заявки и работа клинера.
- `profiles.md` - профили клиента и клинера.
- `push-notifications.md` - регистрация FCM-токена и PUSH-уведомления по заявкам.
- `errors.md` - формат ошибок и основные HTTP-коды.
- `changelog.md` - изменения публичного API.

## Текущие dev-коды

- Клиентский SMS OTP генерируется случайно. Временный код `1111` доступен только при явном включении `KLINOMANIA_CLIENT_OTP_STUB_ENABLED=true`.
- Код клинера: `111111`.

Эти коды являются временными заглушками и управляются через `config/klinomania.php`.
