# Services and checkout API

Все endpoint'ы ниже требуют Sanctum token пользователя с ролью `client`.

- `GET /api/v1/client/home-summary` — число активных заявок и статус ближайшей.
- `GET /api/v1/client/services` — активный каталог в `sort_order`; `id` — стабильный slug.
- `GET /api/v1/client/services/{serviceId}` — карточка услуги, правила цены и варианты конфигурации. Выключенная услуга возвращает `404 service_not_found`.
- `POST /api/v1/client/orders` — оформляет заявку напрямую из услуги и выбранных опций. Заголовок `Idempotency-Key` обязателен и должен быть UUID v4; ключ уникален в пределах клиента.

Заказ не принимает цену, площадь или ID заявки. Backend проверяет конфигурацию, рассчитывает цену как базовый тариф плюс выбранные опции, а также сохраняет снимки услуги, адреса и строк цены. Повтор с тем же idempotency key и тем же нормализованным телом возвращает исходный заказ; другое тело — `409 idempotency_key_conflict`.

Тела запросов и ответов, включая схемы `Service` и `CreateOrderRequest`, зафиксированы в [OpenAPI contract](openapi.yaml). Проверки зоны обслуживания и занятости слота пока не реализованы.
