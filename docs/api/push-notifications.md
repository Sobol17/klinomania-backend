# PUSH-уведомления

## Регистрация устройства

PUSH-уведомления доставляются через Firebase Cloud Messaging (FCM). После
авторизации клиента и при каждом обновлении FCM registration token мобильное
приложение вызывает:

`POST /api/v1/client/push-token`

Endpoint требует Bearer token пользователя с ролью `client`.

Request:

```json
{
  "token": "fcm-registration-token"
}
```

`token` обязателен, имеет тип `string` и максимальную длину 4096 символов.
Успешный ответ — `204 No Content` без тела. Некорректный request возвращает
`422`, неавторизованный запрос — `401`, пользователь не с ролью `client` —
`403`.

Backend хранит только один, последний зарегистрированный токен для клиента.
Повторный запрос заменяет предыдущий токен. Если этот же токен был привязан к
другому клиенту, он переносится текущему пользователю.

## Уведомления о заявках

Статусные PUSH отправляются клиенту после серверных переходов статуса заявки:
`confirmed`, `team_formed`, `in_progress` и `awaiting_payment`. Самостоятельная
отмена заявки клиентом уведомление не отправляет.

Доставка выполняется только если в клиентском профиле включено
`push_notifications_enabled`. Регистрация токена возможна и при выключенной
настройке: приложение может включить разрешение позже через `PATCH
/api/v1/client/profile`.

Уведомление содержит:

- title: `Статус заявки изменён`;
- body: `Заявка {public_id}: {status_label}`;
- FCM data: `{"order_id": "{public_id}"}`.

`order_id` — публичный ULID заявки, который приложение использует для перехода
к её деталям и синхронизации данных через API. Если Firebase сообщает, что токен
недействителен или больше не зарегистрирован, backend удаляет его из профиля.
